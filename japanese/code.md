## 基本
- ### 最小限の機能から実装する
- ### まとまりとの間は改行
- ### 最後は改行
- ### わかりにくい箇所で日本語で「なぜ」がわかるコメントを書く
- ### 正式名所が長ければ省略する

## コード
- ### TypeScript
  - ### どの種類の関数かがわかる名前でエクスポートする
    - ### しかしファイル名は名前のみ
  - ### `export default`はそのまま記述
    - ### しかし基本的にすべて名前付きエクスポート
  - ### `public/files/`に静的ファイルを置く
  - ### @などのエイリアスを使う
  - ### 名前付きエクスポートを使う

- ### React
  - ### ページコンポーネントは最後にPageをつけてエクスポート
  - ### そのほかのコンポーネントは最後にComponentをつけてエクスポート
  - ### 思いモジュールはdynamic importとSuspenseでUX改善

## 命名規則
- ### `.env`などの定数は`UPPER_UNDERSCORES`で記述
- ### プロジェクト名は`kebab-case`
- ### ファイル・フォルダ名は`snake_case`
